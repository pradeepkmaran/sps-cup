Biomedical Signal Processing: Brain Computer Interface (BCI)

Motor Imagery EEG Control

Design a system that decodes motor imagery from EEG signals to allow hands-free control interfaces. Build a signal processing pipeline that classifies multi-channel EEG data based on imagined limb movements, such as left hand versus right hand.

Challenge:

Build a signal processing pipeline that classifies multi-channel EEG data based on imagined limb movements, such as left hand versus right hand. The solution should include preprocessing, like filtering and artifact removal, feature extraction, such as CSP(Common Spatial Patterns) and wavelet transforms, and classification, using methods like SVM(Support Vector Machine) and CNN(Convolutional Neural Network). There are bonus points for real-time implementation or integration with a virtual interface.

Emotional State Recognition

Develop a model that identifies emotional states using EEG signals to support mental health monitoring and user interfaces that adjust to needs. Create a system that processes EEG data to classify emotional states, including happiness, stress, or calmness.

Challenge:

Create a system that processes EEG data to classify emotional states, including happiness, stress, or calmness. Use publicly available datasets, such as DEAP and DREAMER, and apply techniques like frequency band analysis, entropy measures, and machine learning classifiers. Focus on making the results easy to interpret and reliable across different subjects.

Cognitive Workload Estimation

Estimate cognitive workload from EEG signals to improve human-computer interaction and learning systems. Design a pipeline that analyzes EEG recordings to measure cognitive load during tasks, such as problem-solving or multitasking.

Challenge:

Design a pipeline that analyzes EEG recordings to measure cognitive load during tasks, such as problem-solving or multitasking. Include preprocessing, feature engineering, like theta/beta ratio and spectral power, and use regression or classification models. Consider adding real-time feedback mechanisms or integration with adaptive systems.

Wireless Sensing: Wi-Fi Vision

Device-Free Activity Recognition

Use Channel State Information (CSI) extracted from standard Wi-Fi packets to detect and classify human activities in a room (e.g., walking, sitting, waving) without using cameras or wearables.

Challenge:

Use Channel State Information (CSI) extracted from standard Wi-Fi packets to detect and classify human activities in a room (e.g., walking, sitting, waving) without using cameras or wearables. Focus on signal processing techniques for CSI analysis and machine learning models for activity classification.

Environmental & Geospatial Signal Processing

Seismic Event Classifier

Identify earthquakes by clear P and S arrivals, emergent onset, long coda, and strong low-frequency surface waves. Detect explosions through impulsive P waves, weak or absent S waves, short duration, and high-frequency content.

Challenge:

Classify different types of seismic events and distinguish them from urban noise. Identify earthquakes by clear P and S arrivals, emergent onset, long coda, and strong low-frequency surface waves. Detect explosions through impulsive P waves, weak or absent S waves, short duration, and high-frequency content. Recognize urban noise as continuous, high-frequency, repetitive signals without distinct seismic phases.

Acoustic Source Localization

Set up an array of microphones at precisely known positions in the environment. Record the sound signal as it reaches each microphone. Calculate the Time Difference of Arrival (TDOA) of the sound between microphones.

Challenge:

Set up an array of microphones at precisely known positions in the environment. Record the sound signal as it reaches each microphone. Carefully calculate the Time Difference of Arrival (TDOA) of the sound between microphones. Use these time delays to estimate relative distances from the source. Finally, apply triangulation or multilateration techniques to accurately determine and pinpoint the physical location of the sound source.

Deforestation Detector with Radar Imagery

Collect time-series Synthetic Aperture Radar (SAR) satellite images of the target region. Preprocess the imagery to remove noise and align datasets. Compare images across different time intervals to identify significant changes.

Challenge:

Collect time-series Synthetic Aperture Radar (SAR) satellite images of the target region. Preprocess the imagery to remove noise and align datasets. Compare images across different time intervals to identify significant changes in backscatter patterns. Detect anomalies indicating vegetation loss. Highlight and map these areas to pinpoint recent deforestation accurately.

Electronic Vehicles

Inverter Fault Detection

Build a system that detects inverter faults in EVs by analyzing current and voltage waveforms. The pipeline should include preprocessing signals with filtering and denoising methods and extracting features using FFT, Wavelet Transform, and time-frequency analysis.

Challenge:

Build a system that detects inverter faults in EVs by analyzing current and voltage waveforms. The pipeline should include: Preprocessing signals with filtering and denoising methods. Extracting features using FFT, Wavelet Transform, and time-frequency analysis to spot harmonic distortions, short circuits, or switching faults. Using adaptive filtering to separate fault signals from normal noise. Classifying faults with machine learning models for real-time detection. Integrating with onboard vehicle control units for alerts and cloud dashboards for fleet monitoring.

Electric Motor Condition Monitoring

Develop a signal processing pipeline that continuously checks EV motor health using stator current, vibration, and sound emission signals. Use Envelope Detection, Hilbert-Huang Transform, and Spectrogram analysis.

Challenge:

Develop a signal processing pipeline that continuously checks EV motor health using stator current, vibration, and sound emission signals. The system should: Preprocess signals to cut down noise and improve the signal-to-noise ratio. Use Envelope Detection, Hilbert-Huang Transform, and Spectrogram analysis to pull out fault-specific features. Detects issues like bearing wear, rotor misalignment, and insulation breakdown. Implement real-time anomaly classification using AI/ML models like SVM and CNN. Offer predictive insights for preventive maintenance and less downtime.

Road Condition Classification

Develop a road-condition classification system for EVs using vibration signal processing. Acquire vibration signals from accelerometers attached to the suspension and chassis using time-frequency analysis.

Challenge:

Develop a road-condition classification system for EVs using vibration signal processing. The system should acquire vibration signals from accelerometers attached to the suspension and chassis. Use time-frequency analysis, such as STFT and Wavelet Transforms, to classify road conditions like smooth, rough, potholes, wet, or gravel. Connect vibration patterns with tire-road interaction for better suspension control. Enable adaptive ride comfort and proactive safety features like auto-braking and traction control. Support integration with EV fleet management for optimizing routes.
